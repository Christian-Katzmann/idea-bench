# Rejected: Password Sessions Can Trigger AI Spend

Allowing password sessions to call OpenRouter was attractive because it would make the cheapest setup path also the full-featured path, but a password session identifies as the shared literal `operator`, not a person. ïdea Bench blocks password sessions from AI-spending endpoints and requires a GitHub or email identity listed in `AI_ALLOWED_IDENTITIES`, so model spend is tied to an accountable operator.

This should be reconsidered only if the password flow gains per-user identity rather than a shared single-operator credential.
